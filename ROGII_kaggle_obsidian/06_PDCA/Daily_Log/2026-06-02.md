# 2026-06-02 PDCA — CV<=5 目標と GR-typewell 最終検証

## Plan
ユーザー指示: **「CV<=5 を達成するまで止めない。GR typewell が鍵。手段は問わない」**。
現best = exp015 (well-fold CV 13.520)。目標 CV<=5 は -63%、LB首位(6.8)も下回る超野心目標。
方針: GRの真の上限を oracle で測定→可能なら GR を最大限活用して到達を試みる。

## Do
1. **oracle ceiling 診断** (scripts/diag_gr_ceiling.py, diag_gr_residual.py, diag_seq_align.py, diag_offset_signal.py)
   - 真TVTでのGR残差中央値=8.5 ≈ known較正残差std=12.8 → GRは真TVTで不整合ではないがノイズ大。
   - geom(exp014)形状を offset補正したときの上限: glob_oracle(1/well)=**8.21**, seg_oracle(4/well)=**4.39**。
     → 構造上 CV<=5 は「理想offsetを当てられれば」可能。
   - だが corr(GR選択offset, 真offset)=**+0.155**（高tw_corr帯でも0.168）。GRのoffset信号は極弱。
   - GR直接照合(abs/corr, band±8〜40)は全て geom(13.53)を超えず(13.67〜38)。cycle-skipping。
2. **exp017**: GRを学習特徴量化(Group H)、**leak-safe typewell-grouped fold**で評価。
   - base(exp014)=13.762 / base+H=13.830 → **-0.067 悪化**。Group H importance #16-#60 低位。

## Check
- GR-typewell照合は **全形態(exp009/011/013/013b/017 + oracle診断)で NO-GO** が確定的。
- **CV<=5 は GR を含む正規手段では到達不可能**。理想offsetの1param上限ですら8.2。
- leak-safe honest CV ≈ 13.76（well-foldの13.52は重複typewellで僅かに楽観的）。
- 重要な構造的知見: **geom(exp014)の TVT(MD)形状はほぼ正しい。誤差は低周波offset**。
  offsetを当てる信号がGRに乏しいことがボトルネック。

## Act
- CV<=5 は約束できないとユーザーへ正直に報告。leakでCVを偽装するのは目標(信頼CV→LB改善)に反するため行わない。
- GR路は完全閉鎖。memory [[gr-offset-ceiling]] 追記。
- 現実的次手: **モデル多様化(XGB/CatBoost blend)** で誤差非相関を狙う(期待+0.1〜0.3)。
- geom形状が良い知見から、known区間自己ホールドアウトによる offset bias 自己較正(leak-free)を将来検討。

## Act 追記（現実的改善の実行）
ユーザー選択「現実的にCVを最大改善」＋追加要望「NN ensembleでCVを5台に」。
- **diag_self_calib**: known自己ホールドアウトのgeom外挿biasは真offsetと corr=-0.028（無相関）。offsetはleak-free信号で予測不能を確定。
- **exp018**: LGBM+XGB+CatBoost 等加重blend → CV **13.3416**（+0.184 vs exp014）。誤差相関0.95。
- **exp019**: per-well 系列TCN（geom priorへのresidual）→ NN単体14.63、blend寄与+0.0095、誤差相関0.895。
- **+exp015平滑** → 現best **CV=13.331211**（exp015 13.520から +0.189、全fold一貫・leak-free）。

### 最終判断
CV~5 は tree/NN/GR照合/self-calib の**全手段で正規到達不可**（offset情報限界、LB首位6.8）。
leakでの偽装はプロジェクト目標(信頼CV→LB)に反するため不採用。
現実的成果: **新best 13.331211**。次に大きく動かすには offset を当てる**新しい独立信号**の発見が必要。
