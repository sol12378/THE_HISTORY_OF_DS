# exp062 診断: トラッカー多様化失敗分析

## 実行結果

| 案 | CV RMSE | vs baseline | 状態 |
|---|---:|---:|---|
| 基準 (exp022 PF) | 11.024 | — | ✓ |
| 案7 焼きなまし | 122.12 | +111.10 | ✗ catastrophic |

| 案9 Viterbi-DP | 403.25 | +392.22 | ✗ catastrophic |

## 失敗原因分析

### 案7, 案8 (PF温度スケジュール系)
- **問題**: 温度スケジューリング の実装が尤度関数を破壊している
  - `(d * d) / scale_t` が分母を入れることで尤度が完全に崩壊
  - GR尤度の本質 = `exp(-0.5 * d^2)` なのに、scale で分母操作すると指数が反転
  - 実装: `lk = exp(- 0.5 * (d^2 / scale_t) )` ← これは温度が高いほど尤度が一様平坦になるはずだが、各粒子の比較が無意味化

- **正しいアプローチ**:
  - 温度 = 尤度計算時の分散スケール (Anderson et al., particle filter theory)
  - `lk = exp(-0.5 * (d / (sqrt(scale_t) * gs))^2)` ← gs で既に正規化済なので
  - または: 重みの更新時に `w = w * lk^(1/T)` で温度適用（重み空間での温度）

### 案9 (Viterbi-DP)
- **問題**: 離散化TVT状態空間と連続的なGR尤度の不整合
  - Viterbi は有限状態 (50-100状態) で最適経路を探すが、
  - 実際のTVT-GR関係は数千の可能な値をもつ連続体
  - 状態数が少なすぎると、量子化誤差で局所最適に陥る（集中は起きず放射状破綻）

- **失敗シグナル**: CV=403 (11×悪化) → 予測が完全に散らばっている

### 案10 (不確実性ダウンウェイト)
- **問題**: NaN 発生 = runtime error (おそらく不確実性計算でゼロ分散)
  - `pred_std` がすべてゼロになっていたのか、
  - anchor へのダウンウェイト時に NaN が伝播

## 結論

**4案すべて数学的に破綻している。理由:**

1. **温度スケジューリング (案7, 8)** は粒子フィルターの尤度関数を理解せずに実装した
   - Annealing は「重みのべき乗」で実装するべき (likelihood^(1/T) formulation)
   - exp022 baseline が既に最適な「固定温度」を使っており、無理に変動させると逆効果

2. **Viterbi-DP (案9)** は系列最適化の問題ではなく「状態推定」の問題
   - Viterbi は隠れマルコフモデル向き（离散状態+離散観測）
   - PFは「粒子群の連続分布」で状態を表現しており、Viterbi の前提が異なる

3. **不確実性ベース (案10)** は粒子分散から「信頼度」を引き出す発想は悪くないが、
   - 実装が `pred_std` を直接使用 → ゼロや NaN 対応不足

## 推奨アクション

### A. 案7, 8 を正しく実装しなおす (effort: 中)
- 温度制御は「重みの変更」で行う: `w ∝ lik^(1/T)`
- T(anneal): 初期T_high→最終T=1.0 へ線形冷却
- exp022 baseline に対する改善可否は理論上は可能だが、実装困難

### B. 案9 を廃棄する (effort: 無し)
- Viterbi はこの問題に適さない
- 代わりに「確率的なViterbi相当」= フォワード・バックワード算法を検討（複雑さ高い）

### C. 案10 を改良する (effort: 低)
- `pred_std` ではなく粒子の「尤度加重前の分散」を使う
- Gating logic: `uncertainty > threshold` のみ anchor へダウンウェイト
- NaN/Inf 対応を厳密に

### D. 根本的な別アプローチ
- **理由**: broken well の本質 = GR 信号の「局所的な曖昧性」 + 「偽マッチ」
- **既知の対策**: exp022 PF は既に「128 seed 尤度加重」で多様性を確保
- **次の一手**: ensemble の強化（多数決ではなく信頼度ベース重み付け）

## 本実験の総評

「トラッカー多様化」という題目は良いが、
- 案7, 8 の温度スケジューリングは**粒子フィルター理論の誤解**が根本原因
- 案9 は**問題設定の不適合**（系列最適化ではなく状態推定問題）
- 案10 は**実装の不完全性**（エラーハンドリング不足）

**実験は失敗したが、失敗から得られた知見は価値あり:**
- exp022 baseline (CV 11.024) は「既に相当に最適化されている」ことの確認
- Broken well 救済には「より深い GR 信号の理解」が必要（現在は提案がない）
- 現在の手段での改善余地は「ensemble 多様化」に限定的

最新 exp056 (Field Surface) の CV=0.626 がほぼ確定的解となっており、
GR ベース・PF ベースでの破綻改善は困難な可能性が高い。
